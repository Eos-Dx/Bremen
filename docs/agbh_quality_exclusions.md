# AgBH Quality Exclusions

Status: research draft.

Aramis excludes sample sessions whose AgBH calibration context is not reliable
enough for the current K-alpha decision-support dataset. The reason is simple:
if the calibration profile shows K-beta shoulder contamination, wrong q-range
conditions, or another failed calibration-quality criterion, downstream sample
profiles may be shifted or distorted before model training.

The current runtime rule is:

```text
exclude rejected AgBH-linked sessions before GFRM loading
primary key: linked_agbh_session_uid
fallback only for older H5 containers: started_at date
```

The primary key is session-based because one calendar date can contain more than
one calibration session. Date-only exclusion is therefore only a compatibility
fallback when the H5 container does not contain `linked_agbh_session_uid`.

Current machine-readable lists are stored in the Aramis preprocessing YAMLs:

```text
filters.quality_exclusions.primary_key.excluded_values
filters.quality_exclusions.fallback_date.excluded_dates
```
